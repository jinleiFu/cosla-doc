# 注意事项

# 1.新建页面

  1. 根据模块在views文件夹下创建对应文件夹
  2. 根据具体功能创建文件夹，如add、edit、details
  3. 在对应功能下创建index.vue，根据需要判断是否需要创建同级components文件夹
  4. 在router/index.js创建路由
// 路由示例

{
    path: '/vehicle-manage',
    name: 'VehicleManage',
    component: Layout,
    redirect: '/vehicle-manage/details',
    children: [
        {
            path: 'details',
            name: 'VehicleDetails',
            component: () => import('@/views/vehicle/details'),
            meta: {
                title: '车辆详情',
            },
        },
        {
            path: 'add',
            name: 'AddVehicle',
            component: () => import('@/views/vehicle/add'),
            meta: {
                title: '新增车辆',
            },
        },
    ],
}

注意事项：路由添加后访问403可能原因为登录账号没有该页面访问权限

解决方案：

  • 在config/config.js中添加routesWhiteList配置项，设置路由白名单。上线时注意移除该文件的routesWhiteList配置！！！该配置仅在开发阶段使用，线上根据用户权限而定。
  • 请后端小伙伴添加路由权限

# 2.静态资源

项目中使用到的小图标放置assets文件夹，根据模块创建文件夹后放置切图。

# 3.网络请求

所有的网络请求方法均放置在api文件夹中，根据模块命名请求文件，模块中放置该模块所有的请求方法。

请求方法书写示例：

// user.js

import request from '@/utils/request'

/**
 * 获取用户token
 */
export function userToken(tempToken) {
  return request({
    url: `/user/token/${tempToken}`,
    method: 'GET',
  })
}

注意事项：

  • 请求方法都默认添加loading及错误提示，去除该默认行为请见下方示例

    // 去除loading
export function getContractDetails(serviceStationContractId) {
  return request({
    url: `service-station-contract/${serviceStationContractId}/detail`,
    method: 'GET',
    headers: {
      showLoading: false,
    },
  })
}

// 去除错误提示
export function validateServiceStationName(data) {
  return request({
    url: '/service-station/validate/service-station-name',
    method: 'POST',
    data,
    headers: { hideError: true },
  })
}
