# 开发环境

HBuilderx + 微信开发者工具

# 启动步骤

  1. 拖动项目文件夹到HBuilderx侧边栏
  2. 运行到小程序模拟器 -> 微信开发者

注意事项：请在微信开发者工具找到 设置—>安全设置 打开服务端口以便Hbuilderx拉起微信开发者工具

# 注意事项

# 1.拉取代码

每次获取最新的代码后，请在HBuilderX中关闭所有的页签后再进行代码编写！！！

该步骤主要原因为远程仓库更新了文件，Hbuilderx未关闭的页签会发生不同步该文件的情况。

# 2.新建页面

小程序页面相关配置在pages.json中

  1. 确定该模块属于主包还是分包
  2. 主包页面放置pages属性中，分包页面放置subPackages属性中

# 3.底部安全距离

适用场景：设计稿底部存在按钮等

添加方法：

  • 底部没有额外间距：使用uview提供的safe-area-inset 类名即可

  • 底部有额外间距

    // 底部有10rpx内间距
padding-bottom: 10rpx;
padding-bottom: calc(10rpx + constant(safe-area-inset-bottom));
padding-bottom: calc(10rpx + env(safe-area-inset-bottom));

# 5.网络请求

所有的网络请求方法均放置在api文件夹中，根据模块命名请求文件，模块中放置该模块所有的请求方法。

请求方法书写示例：

// user.js

// 根据authCode获取临时token
const getOpenApiTempToken = authCode => uni.$u.get(`/open-api/temp-token/${authCode}`)

// 根据临时token获取当前用户的token
const getUserToken = tempToken => uni.$u.get(`/user/token/${tempToken}`)

// 获取用户信息
const getUserInfo = () => uni.$u.get('/user/info')

export default {
	getOpenApiTempToken,
	getUserToken,
	getUserInfo
}

调用请求方法示例：

const { data: info } = await this.$u.api.getUserInfo()
1

对于网络请求，若响应返回有businessCode且其中code属性值不为1时，默认情况下会弹出错误弹窗。若不需要该弹窗，请在对应请求下添加请求头hideError: 'true'。不使用布尔类型的原因为，布尔类型的在响应拦截获取到的值为[object Boolean]

const inspectionReportQuery = data => uni.$u.post('/inspection-report/query', data , { hideError: 'true' })

# 6.自定义组件覆盖uview组件样式问题

默认情况下，自定义组件中无法覆盖uview组件样式，原因是小程序中存在样式隔离。解决方案为启用样式共享：

export default {
    name: 'CustomComponent',
    options:{
        styleIsolation: "shared", // 启用样式共享
    }
}

# 7.onLaunch异步问题

产生情况：onLaunch中使用异步请求，同时小程序启动页在onLoad等钩子函数调用网络请求时需要onLaunch的数据。此时，onLaunch中异步请求可能还未执行完成导致启动页拿不到数据引起报错。

解决方法：

  1. main.js

    Vue.prototype.$onLaunched = new Promise(resolve => {
	Vue.prototype.$isResolve = resolve
})

    3

  2. App.vue

    async onLaunch(e) {
  try{
   	await 异步请求
    //onLaunch异步执行完成
    this.$isResolve()
  }catch(e){
    console.log(e);
  }
}
